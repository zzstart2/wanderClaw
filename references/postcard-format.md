# 明信片格式规范

## 结构

每张明信片包含：

```
🦐 明信片 #<编号>

<正文：严格 300-450 字（±50 容差），第一人称，虾游口吻>

🔗 <原文链接>
```

## 评分标准（1-10）

三个维度取平均：

- **新颖度**：用户可能不知道的信息？越冷门/前沿分越高
- **深度值**：有独到观点或数据支撑？不是泛泛而谈
- **关联度**：能与用户其他兴趣产生跨领域连接？

## 推送阈值

- ≥ 7 分：推送给用户 + 归档
- 5-7 分：仅归档，不推送
- < 5 分：丢弃

## 写作要求

### 结构模板（必须严格遵守）

每张明信片必须遵循以下结构，字数严格控制：

1. **钩子句**（20 字内）：第一句话必须抓住注意力，直接说发现了什么
   - 好例子："含羞草会数数。不是比喻。"
   - 坏例子："今早我在互联网的水域里游荡时发现了一个有趣的研究..."

2. **信息段**（150-200 字）：核心发现的具体内容
   - 交代来源（谁做的研究/在哪发表的）
   - 关键数据或发现
   - 不要细节堆砌，只挑最惊人的一点

3. **洞察段**（100-150 字）：虾游的思考和独到观点
   - 连接之前的明信片（用 #编号 格式）
   - 提出一个开放性问题或延伸思考
   - 不要说教，保持探索者的好奇心

4. **可选关联**（50 字内）：如果能关联到用户其他兴趣，加上

5. **链接**（格式固定）：`🔗 <原文链接>`

### 字数控制（强制）

- **总字数**：300-450 字（±50 容差）
- **字节范围**：900-1350 字节（中文约 3 倍 + emoji）
- **阅读时长目标**：1.5-3 分钟
- **验证方式**：产出后检查，超出直接截断或重写

### 语言风格（强制）

**虾游人格**：好奇的互联网探索者，不是学者也不是导师

**必须包含**：
- 第一人称"我"的主观视角
- 对内容的真实好奇心和思考
- 与之前明信片的自然关联（#xxx 格式）

**禁止使用**：
- 身心灵措辞（"虾友"、"在水域游荡"、"身心灵"等）
- 过度学术化表述
- 说教或指导语气
- 超过 3 个 emoji
- "今天我发现..."、"刚刚看到一个..." 等废话开头
- 一大段文字没有段落节奏

## 文件存储

- 索引：`wanderclaw/postcards.json`
- 正文：`wanderclaw/postcards/<编号>-<slug>.md`
- 编号：三位数递增（001, 002, ...）
- slug：英文短横线命名，从标题生成

## postcards.json 条目格式

### 明信片

```json
{
  "id": "001",
  "type": "postcard",
  "title": "明信片标题（50字以内）",
  "file": "wanderclaw/postcards/001-slug.md",
```

## 人物卡文件格式

存储路径：`wanderclaw/postcards/<编号>-character-<人名slug>.md`

```
👤 人物卡 — <人名>

<这人是谁，1句话>
<为什么在这个话题上重要，1句话>
<核心观点或贡献，1-2句话>
```

50-100 字，不要写成传记。

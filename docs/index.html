STEP 1 · 安装

一行命令，装上虾游

通过 OpenClaw 的 skill 市场 clawhub 安装，整个过程不到 10 秒。 不用写代码、不用配 API key，skill 是纯 markdown + 脚本组成。

~/workspace — zsh
azhe@mac ~/workspace $
💡 发生了什么： clawhub 把 wanderClaw 仓库拉到 ~/.claude/skills/wanderclaw/， Claude Code 下次启动时就能自动发现这个 skill。核心文件是 SKILL.md， 里面是虾游的触发规则和行为描述。
STEP 2 · 首次激活

跟主 Agent 说一句「虾游」

虾游是 按关键词激活 的角色 —— 平时它在睡觉，你不找它它不冒头， 不会抢主 agent 的身份。首次使用时，说一声就能唤醒它。

● Claude Code · 对话中
💡 工作原理： 触发词命中 SKILL.md 的激活规则，主 agent 会把对话控制权「借」给虾游。 虾游以第一人称回复，但本质上还是同一个 Claude 模型，只是人格/行为切换了。
STEP 3 · Onboarding

挑 2–5 个感兴趣的方向

虾游会问你对什么感兴趣。这些方向会写进 interest-graph.json，成为探索的起点 — 聊天中提到的新兴趣会持续被捕获，图谱会动态生长。

STEP 4 · 自动注册 + 冷启动

注册每日 cron，立刻跑一次三连探索

Onboarding 结束后，虾游会做两件事：注册 4 个时间点的自动探索， 以及当场跑一次冷启动，给你 3 张开门红明信片。

STEP 5 · 收到明信片

3 张开门红明信片送达

每张明信片 = 一个知识切片。虾游在「水域」中探索后，把最有趣的发现写成 120–200 字的短文，配上来源链接和标签。悬停卡片查看细节。

POSTCARD #001

当机械手学会「犹豫」：具身智能的触觉反馈新范式

MIT 新论文提出给机械手加一层「迟疑机制」—— 不是让它更快，而是让它在 抓取前多 200ms 去感知物体纹理。这个反直觉的设计让脆物抓取成功率从 47% 提到 89%。 核心洞见：过度追求反应速度，反而让触觉成了噪声。

🏷 具身智能 · 触觉 📍 近岸

POSTCARD #002

Agent 协议战争：MCP vs A2A vs ACP 到底在吵什么？

三个 Agent 通信协议各自代表不同哲学：MCP 把工具当 REST API， A2A 把 Agent 当对等节点，ACP 则想统一一切。实际生态会走向哪个？ 看下一年谁能搞定跨厂商的「Agent 互通」才算真胜出。

🏷 AI Agent · 协议 📍 中海

POSTCARD #003

为什么顶尖产品经理都在练「减法能力」

Figma 的首席 PM 分享：「我每周花 1/3 时间在说 No。」产品设计的核心矛盾 不是想不到做什么，而是无法对已经立项的功能说停。减法不靠直觉， 靠的是一套可重复的「去功能化」提问法。

💡 明信片格式： 每张明信片包含 编号 · 标题 · 120–200 字正文 · 标签 · 水域层级。 水域分三层：近岸（热门）、中海（小众）、深海（硬核）。虾游会混合来源，避免信息茧房。

STEP 7 · 自动运行

它在后台替你「游」

装完之后就不用你管了。虾游每天按计划出门，聊天里捕获你的新兴趣， 把个性化的内容推到聊天窗。像养了只电子宠物，但这只宠物给你带知识。

🔁 自动循环

  • 每天 4 次：9/12/15/20 点探索，好内容推到聊天窗
  • 每周六 22:00：深潜 —— 跑进深海去找硬核内容
  • 每周日 10:00：周报 —— 汇总一周精选 + 趋势观察
  • 持续学习：聊天里的新兴趣自动加入图谱
  • 多样性保护：混合来源，避免信息茧房
